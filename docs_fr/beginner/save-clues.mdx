- Le second type de clue s'appelle une _Save Clue_.
- À Hanabi, il n'y a qu'un nombre limité de copies d'une carte donnée. Si toutes les copies d'une carte sont défaussées, alors il devient impossible d'obtenir le score maximum. Ainsi, on donne des _Save Clues_ pour communiquer que des cartes spécifiques doivent être sauvegardées pour plus tard.

### Uniquement sur les Cartes Chop

- Les _Save Clues_ **ne peuvent être données que sur des cartes en [chop](./chop.mdx)**.
  - Donc, si une clue est focalisée sur une carte non-chop, alors ce doit être une _[Play Clue](./play-clues.mdx)_ à la place.

### Liste des Cartes à Sauvegarder

- L'équipe **doit sauvegarder** les cartes suivantes (et n'a **le droit de sauvegarder que** les cartes suivantes) :
  - Tous les 5
  - Les 2 uniques (i.e. les 2 dont c'est la seule copie actuellement visible)
  - Les cartes critiques (i.e. les cartes dont toutes les autres copies sont défaussées)
- La raison de sauvegarder les 5 et les cartes critiques est évidente : si elles sont défaussées, le score maximum diminue.
- La raison de sauvegarder les 2 est moins évidente. Nous nous accordons à traiter les 2 comme des cartes critiques car ils sont vraiment importants pour obtenir le score maximum de manière régulière.

### Parfois Vous Ne Saurez Pas, Alors Conservez-La

- Parfois, une clue qui focalise la carte chop pourrait être soit une _[Play Clue](./play-clues.mdx)_, soit une _Save Clue_.
- Quand cela arrive, le joueur recevant la clue doit **considérer les deux possibilités** et conserver la carte jusqu'à être certain.
- Plus précisément, cela signifie qu'il écrira une _[card note](./card-notes.mdx)_ qui inclut toutes les possibilités pour une _Play Clue_ et toutes les possibilités pour une _Save Clue_.

### Trois Types de Save Clues

- Correspondant à la liste ci-dessus, il y a trois types de _Save Clues_ :
  - Le _[5 Save](./5-save.mdx)_
  - Le _[2 Save](./2-save.mdx)_
  - Le _[Critical Save](./critical-save.mdx)_
- Voyons chacun individuellement.
