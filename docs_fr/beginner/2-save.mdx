- Même s'il y a deux copies de chaque 2, nous nous accordons sur le fait que les 2 sont si importants qu'il faut les sauvegarder de manière similaire aux 5.
- Nous nous accordons sur le fait que les 2 doivent être sauvegardés avec **une clue de chiffre 2**. C'est ce qu'on appelle un _2 Save_.
- Dans l'exemple ci-dessous :
  - Alice clue number 2 à Bob, ce qui touche un 2 sur le slot 5.
  - Avant cette clue, Bob n'avait aucune carte cluée dans sa main.
  - Puisque la clue était focalisée sur sa [chop](./chop.mdx), Bob sait que cela doit être un _2 Save_ sur le 2. Cela peut être n'importe quel 2 du jeu.

<TwoSave />

### Quand Plusieurs Copies sont Visibles

- On peut voir le _2 Save_ comme un dernier recours pour sauvegarder un 2. Ainsi, les joueurs n'ont pas le droit d'effectuer un _2 Save_ sur un 2 si ce même 2 est visible dans la main de quelqu'un d'autre.
- C'est parce que l'équipe n'a en réalité pas encore besoin de sauvegarder ce 2 – elle pourra sauvegarder l'autre copie plus tard.
- Par exemple, dans une partie à 3 joueurs :
  - Bob a un red 2 sur le slot 5 (sa chop).
  - Cathy a un red 2 sur le slot 3.
  - Ainsi, il serait illégal pour Alice d'effectuer un _2 Save_ sur le red 2 de Bob.
